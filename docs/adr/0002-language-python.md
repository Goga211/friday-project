# ADR 0002 — Язык Hub/Core/агентов: Python

Статус: принято (Phase 0). Дата: 2026-07-01.

## Контекст
Возник вопрос: не будет ли Python медленным для такого проекта?

## Решение
**Python 3.12 (asyncio)** для Hub/Core и агентов.

Обоснование:
- Система **I/O-bound**: узкое место — облако (STT, Claude) и сеть, а не скорость языка.
  Накладные расходы Python на сообщение — доли мс против ~100–300 мс сетевого RTT.
- «Горячие» локальные пути (wake-word ONNX, аудио, Piper TTS) — **нативные библиотеки**,
  Python только дирижирует. GIL не мешает: asyncio + либы отпускают GIL.
- Экосистема ИИ/голоса **Python-first**: Anthropic SDK, Whisper, openWakeWord, Piper,
  LangGraph; Home Assistant сам на Python.
- Скорость разработки для соло-проекта.

## Полиглотность
Шина (MQTT) язык-независима → отдельный компонент можно позже переписать на Go/Rust
(напр. тончайший агент-исполнитель), не трогая остальное. Сейчас — YAGNI, всё на Python.

## Последствие
Не писать тяжёлый DSP/обработку звука чистым Python — только через нативные либы.
